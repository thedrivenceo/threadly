<Update label="2026-04-13" description="v2.0.7" tags={["Improvement", "Fix"]}>
  ### App Renamed to Post Companion

  The app has been renamed from PublishMuse to Post Companion. All documentation, UI references, and branding updated to reflect the new name.

  ### Analytics: Top Views and Post Timestamps

  - **Top Views section**: Analytics now shows a "Top Views" row displaying the 3 most-viewed posts, alongside existing Top Likes, Top Comments, and Top Shares
  - **Post timestamps on analytics cards**: Every post card in the top posts sections now shows the date and time it was posted (e.g. "Apr 10, 12:05 PM")

  ### Engagement Reply Fix

  - Fixed a bug where clicking "Reply" on the Engagement page created a new standalone post instead of replying in the thread
  - The `reply_to_id` was incorrectly using the commenter's reply ID (which the user doesn't own) instead of the user's original post ID
  - Replies now correctly post as thread replies and auto-prepend @username to direct the response

  ### Engagement Replies Nest Under Correct Comment

  - Replies from the Engagement tab now thread directly under the specific comment being responded to, instead of appearing as a top-level reply on the main post

  ### Self-Replies Filtered from Engagement Tab

  - Threadstorm reply chains and your own replies no longer appear in the Replies tab
  - Only comments from other users are shown

  ### Pending Review Status on Create Page

  - Posts can now be saved with a "Pending Review" status for team review before publishing
  - Pending posts appear in the Content Bank with a yellow badge
  - Pending posts can be approved or edited before going live
</Update>


<Update label="2026-04-08" description="v2.0.6" tags={["Improvement"]}>
  ### Create Page Scheduling Improvements

  - **Best Slot** saves, clears the form, and stays on Create page for rapid content creation
  - **Use best time** now auto-schedules in one click (picks best hour from engagement data for your selected date)
  - **Schedule** clears the form and stays on Create page instead of navigating away
  - **Calendar post navigation**: prev/next arrows in edit dialog to move between posts without closing
  - Fixed build error that was preventing deploys
</Update>

<Update label="2026-04-08" description="v2.0.5" tags={["Fix"]}>
  ### Reply Tracking and Auto-Dismiss

  - Daily reply counter now tracks actual replies sent from the Engagement tab
  - Replying to a comment auto-dismisses it (no need to click the checkmark after)
  - Checkmark is now only for "nothing to say" dismissal
  - Dashboard shows real reply count instead of placeholder data
  - Counter resets daily
</Update>

<Update label="2026-04-08" description="v2.0.4" tags={["Feature"]}>
  ### Timezone Support

  - **Timezone selector** in Settings with 30 common timezones
  - Auto-detected on first account creation
  - **Auto-publish** now respects each user's timezone when deciding if a post is due
  - **Webhook** best time calculation uses your timezone, not UTC
  - No more posts publishing at the wrong time
</Update>

<Update label="2026-04-08" description="v2.0.3" tags={["Fix"]}>
  ### Engagement: Direct Replies and Persistent Dismissals

  - You can now **reply directly** to comments from the Engagement tab. Replies are posted via the Threads API.
  - **Dismissed replies persist** across devices and browser clears. Previously stored in the browser, now saved in Supabase.
  - Replying to a comment auto-dismisses it after posting.
</Update>

<Update label="2026-04-07" description="v2.0.2" tags={["Feature"]}>
  ### Pending Approval Workflow

  Posts sent via webhook with a date now arrive as **pending** (amber) instead of immediately scheduled. This gives you a chance to review and edit before committing.

  - Pending posts appear in amber on the calendar and in the Content Bank
  - Click **Approve** to schedule, or edit first then approve
  - Auto-publish only processes approved (scheduled) posts
  - Webhook still accepts `"status": "scheduled"` to skip approval if needed
  - Webhook auto-fills best posting time when date is provided without a time
</Update>

<Update label="2026-04-07" description="v2.0.0" tags={["Major Release"]}>
  ### Post Companion Launch (formerly PublishMuse)

  Complete redesign and feature overhaul. Formerly Threadflow, now Post Companion.

  **Visual Overhaul**
  - Threads-inspired black and white theme with colored accent icons
  - Figtree font across the entire app
  - True black dark mode (#000), pure white light mode (#fff)
  - Solid borders, shadow card system, hover lift effects
  - Green activity heatmap, solid calendar post colors
  - Theme toggle in top right corner

  **Multi-Account Threads Support**
  - Connect multiple Threads accounts (Free: 1, Pro: 3, Team: 10)
  - Switch between accounts on the Accounts page
  - All edge functions use the active account

  **AI Content System**
  - Custom AI prompts page for repurposing and audio processing
  - Dynamic prompts built from brand voice, writing samples, templates, and categories
  - Brand voice document upload as alternative to questionnaire
  - Three output formats: short (1-2 sentences), mid (1 paragraph), long (multi-thread)
  - AI rate limiting: 10/day free, 100/day pro and team

  **Smart Scheduling**
  - Posting frequency setting (1-5x per day) on Goals page
  - Calendar and list view show placeholder slots at best times from engagement data
  - "Best slot" button auto-schedules to the next available slot
  - "Schedule" button on cards opens date picker with auto best time
  - Best slot available on Create, Content Bank, and Inspiration pages

  **Content Bank Improvements**
  - Pillar and content type selectors on each card
  - Dual filter dropdowns (pillars and content types)
  - Inline Best Slot, Schedule, and Delete buttons on every card
  - Solid borders replacing dashed borders

  **Categories Page**
  - Two-column layout: Content Types and Content Pillars side by side
  - Inline edit for name and color on each item

  **Goals Page Redesign**
  - Follower goal with celebration (confetti + toast when reached)
  - Daily replies as dropdown selector
  - Posting frequency setting with green icon
  - All cards with colored icon containers

  **Chrome Extension**
  - Save inspiration from any page via popup
  - One-click bookmark button on Threads posts
  - Pillar and content type selection in popup
  - Connection test on setup with clear error messages
  - Categories fetched fresh from server on every open

  **Navigation Reorganization**
  - AI Prompts and Accounts as standalone pages
  - Settings menu (UserMenu popover): My Profile, Team, Billing, AI Prompts, Accounts, Admin
  - Sidebar: core content pages only (Dashboard through Goals)
  - 404 redirects to dashboard (logged in) or sign in (logged out)

  **Security Hardening**
  - Per-user webhook secrets (unique per account, not shared)
  - AI generation rate limiting (10/day free, 100/day pro)
  - Admin emails controlled via environment variable
  - CORS restriction support via ALLOWED_ORIGINS secret
  - Email verification required
  - Strong password policy: uppercase, lowercase, digits, leaked password check

  **Data Migration**
  - All user data moved from localStorage to Supabase
  - Style guide, brand doc, goals, templates, writing samples all persist across devices
  - Only theme preference and dismissed replies remain local

  **Infrastructure**
  - Vercel SPA routing (no more 404 on page refresh)
  - 19 edge functions (added: threads-switch-account, save-inspiration)
  - New tables: user_settings, user_templates, user_writing_samples, user_prompts
  - PWA manifest and service worker updated

  **Billing Page**
  - Larger feature text (14px)
  - Fixed "Most Popular" badge contrast in dark mode
  - Webhook examples clarified (draft vs scheduled, with category/type)
</Update>
